# Phasemill v1.2.0

Phasemill 1.2 makes required Pi review reliable in autonomous `codex exec`
runs and hardens the CLI fallback used when the bundled MCP transport is not
available.

## Highlights

- persist project-owned approval for sending review context to Pi without
  pausing every autonomous run
- run Pi with direct networking and an isolated sandbox-writable config while
  reusing only the stored ZAI API credential
- preserve provider, model, upstream error, bounded stderr, and streaming
  progress diagnostics when external review fails
- accept long controller results through `--result-file` and reject missing
  stdin within one second instead of stalling indefinitely

See [CHANGELOG.md](https://github.com/krajcik/phasemill/blob/v1.2.0/CHANGELOG.md)
for the full compatibility and safety notes.
